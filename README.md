# 지능형 전자결재 시스템 — 와이어프레임

> 리스크 기반 차등 승인 및 HTML/JSON 분리 아키텍처를 도입한 그룹웨어 전자결재 시스템 와이어프레임입니다.

> 기안자·결재자·관리자 역할 분리, 고위험 문서 2FA 검증, 부서별 동적 양식 빌더, 방어적 설계(Defensive Design) 원칙을 화면 단위로 구현합니다.

## 1. 프로젝트 소개

이 프로젝트는 실제 기업 그룹웨어(HaiWorks, 다우오피스, Naver Works 등)를 벤치마킹하여 설계한 **전자결재 시스템 와이어프레임**입니다.

### 핵심 설계 원칙

| 원칙 | 설명 |
|---|---|
| **리스크 기반 차등 승인** | 기안 금액이 설정 임계값을 초과하면 자동으로 고위험 결재 프로세스(2FA 인증 필요, 결재자 전체 열람 강제)로 전환됩니다. |
| **HTML/JSON 분리 저장** | 결재 본문은 사용자에게 보이는 HTML 스냅샷(`content`)과 시스템이 처리하는 구조화 데이터(`document_data` JSON)를 분리하여 저장합니다. |
| **방어적 설계 (Defensive Design)** | 회수 최소 10자 입력 조건, 스크롤 완료 검증, OTP 2FA, 422 에러 코드 명시 등 사용자 실수와 시스템 오류를 사전 차단하는 UI를 각 화면에 적용합니다. |
| **역할 분리 라우팅** | 기안자(박도윤) · 결재자(김기훈) · 관리자(부서 관리) 3개 역할별로 레이아웃과 접근 경로가 완전히 분리됩니다. |

## 2. 사전 준비물 (Prerequisites)

코드를 실행하려면 아래 두 가지 프로그램이 컴퓨터에 설치되어 있어야 합니다.

### Node.js (필수)

JavaScript를 실행시켜 주는 런타임 환경입니다. 반드시 **LTS(Long-Term Support)** 버전을 설치하세요.

1. 브라우저 주소창에 **`https://nodejs.org`** 를 입력하여 공식 사이트에 접속합니다.
2. 초록색 **"LTS"** 버튼을 클릭하여 설치 파일을 내려받습니다.
3. 내려받은 파일을 실행하고, 모든 설치 단계에서 "Next(다음)"를 클릭해 기본값으로 설치합니다.
4. 설치가 완료되면 터미널(아래에서 여는 방법 안내)을 열고 `node -v`를 입력했을 때 버전 번호(예: `v20.17.0`)가 표시되면 성공입니다.

### Visual Studio Code (권장 편집기)

코드 편집기이자 이 가이드의 기준 환경입니다.

1. 브라우저 주소창에 **`https://code.visualstudio.com`** 을 입력합니다.
2. 파란색 **"Download"** 버튼을 클릭해 설치합니다.

## 3. 터미널 열기 및 프로젝트 폴더 이동

터미널(Terminal)은 컴퓨터에 명령어를 직접 입력하는 검정 화면입니다. 겁먹지 않아도 됩니다. 아래 순서대로만 따라 하세요.

### VS Code로 소스코드 폴더 열기

1. VS Code를 실행합니다.
2. 상단 메뉴에서 **파일(File) → 폴더 열기(Open Folder…)** 를 클릭합니다.
3. 다운로드한 프로젝트 소스코드가 들어 있는 폴더를 선택하고 **열기(Open)** 를 클릭합니다.
   - 예시: `C:\Users\홍길동\Downloads\전자결재시스템` 같은 경로의 폴더를 선택합니다.
4. VS Code 왼쪽 탐색기 패널에 프로젝트 파일들이 나타나면 성공입니다.

### VS Code 내장 터미널 열기

방법 1 — 메뉴 클릭:
> 상단 메뉴에서 **터미널(Terminal) → 새 터미널(New Terminal)** 클릭

방법 2 — 단축키:
> **Windows**: `` Ctrl + ` `` (백틱 키, 숫자 1 왼쪽 키)
> **macOS**: `` Ctrl + ` ``

화면 아래쪽에 터미널 창이 열리고 커서가 깜빡이면 준비 완료입니다.

> **확인:** 터미널에 현재 폴더 경로가 프로젝트 폴더 경로와 같은지 확인하세요. 다르다면 아래 명령어로 이동합니다.
> ```bash
> cd 프로젝트폴더경로
> # 예시 (Windows): cd C:\Users\홍길동\Downloads\전자결재시스템
> # 예시 (macOS):   cd ~/Downloads/전자결재시스템
> ```

## 4. 패키지 매니저 pnpm 설치

이 프로젝트는 **pnpm**이라는 패키지 매니저를 사용합니다. npm보다 빠르고 디스크를 적게 차지합니다. 터미널에 아래 명령어를 **한 줄씩** 붙여넣고 Enter 키를 누르세요.

```bash
npm install -g pnpm
```

> **명령어 설명:** `npm install -g pnpm`
> - `npm`: Node.js와 함께 자동 설치된 기본 패키지 도구입니다.
> - `-g`: 컴퓨터 전체에서 사용할 수 있도록 "전역(global)" 설치합니다.
> - `pnpm`: 설치할 패키지 이름입니다.

설치 후 `pnpm -v`를 입력하면 버전 번호(예: `9.15.0`)가 출력됩니다.

### 권한 오류가 날 때

#### macOS — `EACCES` 오류

```bash
sudo npm install -g pnpm
```

비밀번호 입력 화면이 나타나면 Mac 로그인 비밀번호를 입력하고 Enter를 누릅니다. (비밀번호를 입력해도 화면에 아무것도 표시되지 않는 것이 정상입니다.)

#### Windows — 권한 오류

1. 시작 메뉴에서 **"PowerShell"** 을 검색합니다.
2. **"관리자 권한으로 실행"** 을 클릭합니다.
3. 열린 창에 `npm install -g pnpm` 을 입력하고 Enter를 누릅니다.

## 5. 의존성 패키지 설치

프로젝트가 동작하려면 수십 개의 외부 라이브러리(패키지)가 필요합니다. 아래 명령어 한 줄로 모두 자동으로 설치됩니다.

```bash
pnpm install
```

> 터미널에서 파일들이 쭉 내려가며 설치되는 화면이 보입니다. 인터넷 속도에 따라 1~3분 정도 소요됩니다. 마지막에 `Done in X.XXs` 또는 `dependencies` 관련 메시지가 나오면 완료입니다.

설치가 끝나면 프로젝트 폴더 안에 `node_modules`라는 폴더가 생성됩니다. 이 폴더는 자동 생성된 것이므로 건드리지 않아도 됩니다.

## 6. 로컬 서버 실행 및 브라우저 접속

모든 준비가 끝났습니다! 아래 명령어를 입력하면 개발 서버가 시작됩니다.

```bash
pnpm dev
```

터미널에 아래와 비슷한 메시지가 나타납니다:

```
  VITE v6.x.x  ready in 500 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.x.x:5173/
```

### 브라우저로 접속하기

1. **Chrome** 브라우저를 엽니다. (Edge, Firefox도 됩니다)
2. 주소창을 클릭합니다.
3. 터미널에 표시된 `http://localhost:5173` 주소를 **복사해서 붙여넣기** 합니다.
4. Enter를 누르면 전자결재 시스템 로그인 화면이 열립니다.

> **서버 종료 방법:** 터미널에서 `Ctrl + C` 를 누르면 서버가 종료됩니다.

> **다시 시작:** 터미널에서 `pnpm dev` 를 다시 입력하면 됩니다.

## 7. 주요 화면 목록

로그인 후 아래 화면들을 탐색할 수 있습니다.

| 역할 | 화면 | 경로 | 설명 |
|---|---|---|---|
| 기안자 | 결재 작성 | `/` | Rich Text 에디터로 기안 작성, 상신 확인 모달 |
| 기안자 | 내 기안함 | `/drafts` | 기안 목록 조회 |
| 기안자 | 기안 상세 | `/drafts/:id` | 기안 상세 보기, 회수 기능(최소 10자 사유 입력 조건) |
| 결재자 | 결재 대기함 | `/pending` | 대기 중인 결재 목록, 고위험 차단 경고 모달 |
| 결재자 | 고위험 결재 | `/pending/:id` | 스크롤 완료 검증 + OTP 2FA 인증 화면 |
| 관리자 | 위험도 기준 | `/dept` | 고위험 분류 금액 및 조건 설정 |
| 관리자 | 양식 빌더 | `/dept/forms` | 5종 문서 양식 템플릿 관리 및 결재선 설정 |

| 역할 | 이름 | 비고 |
|---|---|---|
| 기안자 | 박도윤 | 사원 / IT 기획팀 |
| 결재자 | 김기훈 | 과장 / IT 기획팀 |
| 관리자 | (부서 관리) | 관리자 메뉴에서 접근 |
